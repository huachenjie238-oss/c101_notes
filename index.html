Clay 完整功能手册

Clay 是一个 B2B 自动化获客平台。把它想象成一张超级表格——它不只能存数据，还能自动联网抓数据、用 AI 处理数据、再把结果推送到你用的所有销售工具里。本文将拆解每一个功能，并附上真实使用场景。

F · Find — 找人找公司

6 个功能
🏢
Find Companies · 批量找目标公司
按行业、规模、地区等条件，直接从 Clay 内置数据库拉取公司名单
›
是什么
就像在网上搜公司，但比搜索引擎精准 100 倍。你设定条件，Clay 自动帮你找出所有符合要求的公司，直接放进你的表格里，不需要手动复制粘贴。
可以过滤的条件
行业 Industry 员工人数 Headcount 所在城市 / 国家 公司技术栈（用了什么软件） 融资情况 成立年份
操作步骤
1
新建一个 Workbook（工作簿），然后在里面新建一张 Table（表格）
2
选择数据源为 Find Companies
3
依次填写你的条件：行业选 Software Development，人数选 500+，地区选 New York
4
点击 Preview 预览结果数量（比如 162 家），满意后点击 Import 导入
5
⚠️ 导入时弹出的 Enrichment 选项全部不勾选，直接以基础数据导入（基础导入免费）
真实使用场景
场景：卖 B2B SaaS 的销售代表 SDR / BDR
你卖的是一款帮销售团队管理客户数据的软件，目标客户是纽约的中大型软件公司。你设定：行业 = Software Development，人数 = 500+，城市 = New York。30秒后，Clay 给你拉出 162 家公司的名单，每家都有公司名、官网域名、LinkedIn 链接。这就是你这个月要攻下的目标池。
⚠ 导入时 Clay 会问你要不要自动抓取额外数据（技术栈、新闻等），新手不要勾选，这会迅速消耗你的试用积分。先把基础名单拿下来，后面再一步步丰富。
👤
Find People · 找公司里的具体决策人
基于公司名单，找出每家公司里你想联系的人
›
是什么
你已经有了一批目标公司，但公司不会回你邮件——得找到具体的人。这个功能就像 LinkedIn 高级搜索，但批量对所有目标公司同时执行，找出每家公司里符合条件的决策人。
可以过滤的条件
职位名称关键词（VP / Director / Head） 级别 Seniority 所在城市 在当前职位任职时长 LinkedIn 简介关键词 持有 HubSpot / Salesforce 认证
操作步骤
1
在公司表格右上角点击 Actions → Find People at these companies
2
填写职位关键词：比如 Revenue、GTM、Sales、Marketing
3
勾选级别：只找 VP、Director、Head（C-suite 也可加上）
4
预览人数（如 753 人），点击 Continue for free 导入基础名单
重要机制：两张表自动绑定
Find People 会自动生成一张新的"人员表"，这张表和你的"公司表"天然关联。如果你后来在公司表里查了新数据（比如该公司用什么技术栈），人员表里点一下刷新，每个人的档案里自动出现这个新数据——不需要做任何 Excel 里的 VLOOKUP。
真实使用场景
场景：找 162 家公司的销售高管 OutBound 获客
你从 162 家纽约软件公司的名单出发，设定条件：职位含 Sales / Revenue，级别是 VP 或 Director 以上，地区在纽约。Clay 在这 162 家公司里挖出了 753 个符合条件的人，每人都有姓名、职位、公司、LinkedIn 链接。这就是你的触达对象名单。
📍
Google Maps 本地商家搜索
专门针对线下实体店、本地服务商，不靠 LinkedIn
›
是什么 / 为什么需要它
很多行业的客户根本不在 LinkedIn 上，比如牙科诊所、水管工、餐厅、健身房。这时候 LinkedIn 搜索没用，但 Google Maps 有这些商家的完整数据。Clay 直接接了 Google Maps 的接口，你可以用地图圈定一个区域，抓出该区域里的所有目标商家。
操作步骤
1
新建表格，数据源选 Google Maps
2
在地图上选定位置和搜索半径（可以精细到某个街区）
3
选择搜索类型：推荐用 Business type search，输入 "dentists" / "plumbers" 等类别
4
等待数据加载（数据会逐渐增多，别太早看），最终得到该区域所有匹配商家，含名称、地址、评分、电话、网站
真实使用场景
场景：向牙科诊所卖诊所管理软件 本地商家获客
你在卖一款帮诊所管理预约和患者档案的软件。在 Google Maps 里选定布鲁克林，搜索类型填 "dentists"，搜索半径设为 5km。Clay 拉出 315 家诊所，每家都有名字、地址、电话、网址、Google 评分和评论数。接着你用 Claygent（AI 爬虫）去抓每家诊所的网站，提取医生人数、开业年份、提供哪些治疗项目，用于个性化开发信。
ℹ Google Maps API 每次搜索最多返回 1000 条记录。如果你的目标区域太大，分批次搜索，比如按城区分拆。
💼
Find Job Signals · 用招聘信号找目标客户
一家公司在招什么人，就暴露了他们的痛点和预算
›
为什么招聘信息是黄金信号
公司招人 = 公司在花钱解决问题。如果你能在招聘信息里找到你的目标客户的痛点，你的开发信就有了完美的切入口。比如一家公司在大量招聘销售代表，说明他们在扩张销售团队——这时候卖给他们"销售自动化工具"，切入点就是"我看到你们在招 BDR，我们的工具能让每个 BDR 的效率翻倍"。
两种玩法
玩法A
以职位为起点找公司：新建表格，Sources → Find Jobs，搜索职位名称（如 bdr、sdr、Business Development Representative），过滤国家和类型，得到一批正在招这类人的公司，再从公司找到决策人发邮件
玩法B
给已有公司名单查招聘数量：在已有的公司表格里，添加 "Find Jobs Waterfall" 集成，查每家公司目前开放了多少职位，用招聘数量做筛选条件，只联系"正在积极招人"的公司
真实使用场景
场景：卖销售自动化工具 Intent-based 获客
你搜索 "bdr sdr sales development representative"，地区限定美国，只看全职岗位，得到 500 个正在招 BDR 的公司。这些公司的邮件开场白写："我注意到贵公司最近在积极招聘 BDR——我们的工具能帮你们的销售团队把每个 BDR 的触达效率提升 3 倍，同时节省 70% 的数据整理时间。" 因为有具体信号，这封邮件的回复率远高于泛泛的推销。
🔍
Company Lookalikes · 找相似公司
基于你最好的现有客户，找和他们长得最像的潜在客户
›
是什么
你有 10 个最满意的客户，这个功能能帮你找出全球范围内和这 10 家公司最相似的公司（行业、规模、商业模式都相近）。逻辑是：买你产品的人已经证明你的产品对他有价值——那么找和他长得最像的公司，转化率也会更高。
操作步骤
1
在已有的公司表格里，添加 Ocean.io 相似公司 集成，以公司域名作为查询依据
2
设置过滤条件：国家（如 United States）、规模（如 200人+）、行业（如 software）
3
点 Save and run，系统为每家原始公司返回 10 家相似公司（162家 → 最多 1620 条新线索）
4
用 Write to Table 将嵌套的数据展开到新表格，再对域名列做去重（dedup），得到干净的新名单
真实使用场景
场景：基于现有客户扩张，并写出杀手级邮件 社会证明策略
你有 20 个 Marketing Agency 客户，用 Lookalikes 找出 200 家类似的营销机构。给这 200 家发邮件时，你可以写："我们帮助了和你们规模相近的 [现有客户名]，帮他们把 outbound 效率提升了 40%。我觉得你们的情况非常类似，想聊聊？" 因为提到了同行的真实案例，这种邮件的回复率是普通邮件的 2-3 倍。
📂
外部数据导入 · CSV / CRM
把你已经有的名单导入 Clay，再进行数据增强
›
支持的导入方式
CSV / Excel 文件 HubSpot 联系人列表 Salesforce（最多2000条 / 用 List 而非 Report） Close CRM Webhook 实时推送
HubSpot 导入注意事项
1
先在 HubSpot 里建好目标列表（比如筛选出所有付费客户）
2
在 Clay 里选 Sources → Import from HubSpot，选择对应列表
3
建议勾选 exclude empty properties，避免导入一堆空列把表格弄乱
4
导入后数据以 Object 形式折叠在一列里，点击单元格提取你需要的字段（邮箱、姓名等），生成独立的列
真实使用场景
场景：对老客户做复购营销 客户成功
你从 HubSpot 导出所有上一季度购买过但 60 天没续费的客户列表，导入 Clay。然后用 AI 爬取他们公司最近的新闻（有没有融资、有没有扩招），根据最新情况生成个性化的挽回邮件，比泛泛的"我们想了解您的使用情况"邮件精准 10 倍。

E · Enrich — 数据增强

3 个功能
💧
Waterfalls · 瀑布流多数据源串联
Clay 最核心的省钱机制——多个数据商串联查询，找到即停
›
原理（用大白话说）
想象你要找一本绝版书。你先去最近最便宜的书店找，找到了就结束；找不到再去第二家书店，以此类推。瀑布流就是这个逻辑——把多个数据供应商排成一列，从最便宜的开始查，查到了立刻停止，没查到才流向下一个。这样最贵的供应商只处理最难查的数据，极大地节省费用。
真实运行示例：查公司融资阶段
Intellizence 1 积分/条
✓ 查到 3 家 → 停止
Dealroom 4 积分/条
✓ 接手剩余7家 → 查到5家
Harmonic 4 积分/条
✓ 查到1家
Pitchbook 3 积分/条
✓ 查到最后1家
Owler 10 积分/条
— 一次都没触发
不管从哪个供应商查到，最终结果都汇总在同一列"Funding Stage"里，干净整洁。
两大额外福利
福利1
找不到就退款：某个供应商没查到数据，Clay 自动退还该次查询消耗的积分。你只为"真正找到的数据"付钱。
福利2
无需单独购买数据库会员：Pitchbook、Clearbit 这些数据库单独购买年费可能要数万美金。Clay 已经和他们谈好了合作，你只需要有 Clay 的订阅，就能调用他们的数据。
⚠ 配置好瀑布流后，不要直接点"全部运行"！新手务必先选"Save and run 10 rows"，小批量测试结果是否符合预期，再全量跑。
📧
Work Email Waterfall · 找真实邮箱
用多个供应商串联找工作邮箱，自动验证真伪，找不到退积分
›
是什么
找到了人，但没有联系方式，等于白找。Work Email Waterfall 会通过多个邮箱数据商，用对方的姓名 + 公司域名 + LinkedIn 链接来推算并验证他的真实工作邮箱，并自动调用 ZeroBounce 等工具验证邮箱是否真实存在。
最优原材料组合（准确率从高到低）
✓ 最优：姓名 + 域名 + LinkedIn 链接 次优：姓名 + 域名 兜底：仅域名（准确率最低）
关于 Catch-all 邮箱
有些公司设置了"Catch-all"——不管发给什么邮箱地址，邮件都会被接收，所以系统无法确认这个地址是否真实存在。如果你非常在意发件域名的安全性（怕触发垃圾邮件过滤），可以关闭 Catch-all 选项，只保留确认存在的邮箱。
兜底策略：私人邮箱 + 条件运行
如果工作邮箱死活找不到，可以添加一个"个人邮箱瀑布流"作为兜底。配合 Conditional Runs，设置"仅当工作邮箱列为空时，才查个人邮箱"——这样不浪费积分查那些已有工作邮箱的人。
真实使用场景
场景：给 753 个决策人找邮箱 Cold Email
你的 753 人名单里只有 LinkedIn 链接，没有邮箱。运行 Work Email Waterfall 后，Clay 依次调用各供应商，最终找到约 600+ 个经过验证的工作邮箱（带绿色勾勾标记）。找不到的 150 人继续跑个人邮箱瀑布流兜底，再多覆盖一部分。整个过程全自动，你喝杯咖啡的时间就完成了。
📊
Enrich Company / Person · 标准数据填充
只要有域名或 LinkedIn 链接，一键拉出所有基础字段
›
Jigsaw（拼图）框架 — 数据的先后顺序
Clay 有一个核心逻辑：找数据就像拼拼图，必须先有"四个角"，才能向外扩展。没有角，什么都拼不了。
数据类型必须先有（角）然后才能得到（边 / 中间）
公司数据 公司域名 + LinkedIn 主页链接 员工数、行业、融资、地址、技术栈、近期新闻……
个人数据 LinkedIn 主页链接 + 全名 职位、简介、教育背景、工作经历、联系方式……
只有公司名怎么办
补救
用 Google Search 集成或 Claygent（AI爬虫），输入"公司名称"，让它反向查出该公司的域名和 LinkedIn 主页链接，有了这两个锚点再继续丰富

T · Transform — AI 处理与转化

6 个功能
🤖
Claygent · AI 网页爬取代理
能上网的 AI 实习生——你问什么，它去网上找什么
›
是什么
Claygent 是 Clay 最强大的 AI 工具。它把"网页爬虫"和"大语言模型"结合在一起：你给它一个问题，它会自动打开浏览器、搜索网页、阅读内容、提取答案，最后返回结构化数据填进你的表格。任何在互联网上公开存在的信息，它都能找到。
能问它的问题举例
这个创始人毕业于哪所大学？ 这家公司最近参加了哪些 Webinar？ 这家公司的网站最后是什么时候更新的？ 这个 CEO 在 Twitter 上最近发了什么内容？ 这家公司的年报里提到了什么战略目标？
模型选择策略（省钱版）
模型适合场景成本
Helium（Clay自研）简单的数据提取，网页结构清晰最低，优先用
Neon / ArgonHelium 抓不准时升级中等
GPT-4o / Claude Opus极复杂推理，长页面深度分析最贵，最后用
Column Output Definition — 结构化输出
Claygent 默认返回一大段文字。但你可以提前定义好"把结果拆分到哪几列"，比如查一家公司的融资情况，让它自动分成四列：
列A：融资阶段 列B：融资金额 列C：领投机构 列D：业务摘要
这样抓回来的数据直接是结构化的，可以直接对接 CRM 或邮件发送工具，不用再二次处理。
真实使用场景
场景：为每个决策人生成个性化邮件钩子 超级个性化
你让 Claygent 去查每个联系人的 LinkedIn 主页，提取他们最近发布的帖子主题或转发内容。如果对方最近分享了一篇关于"AI 在销售中的应用"的文章，你的邮件破冰句就可以写："看到您最近转发了关于 AI 在销售领域应用的内容——我们刚好做了一个案例，帮助一家同类公司用 AI 把响应速度提升了 60%。"

S.P.I.C.E. 框架 · 写出好 Prompt 的模板
Clay 官方推出的 Prompt 写作框架，解决"批量跑时 AI 乱写"的问题
›
为什么 Clay 里的 Prompt 不能随便写
在 ChatGPT 里写的 Prompt，不满意可以让它重新写。但在 Clay 里，一条 Prompt 要同时在 753 个人的数据上运行，没有人工修改的机会。一旦写得不好，753 条结果都是废话。所以需要一个结构化框架，确保第一次就写对。
S.P.I.C.E. 五个要素
S
Sections · 区块划分
用 # 号分隔不同部分，如 # VARIABLES、# INSTRUCTIONS，让 AI 清楚哪里是指令、哪里是数据
P
Prompt Variables · 变量
用 {花括号} 引入表格里的动态数据，如 {full_name}、{company_domain}，每行数据自动替换成对应的值
I
Instructions · 明确指令
给出分步骤的、非常具体的操作指南，如"先查有没有获奖，没有再查有没有晋升，还没有就查认证"
C
Context · 背景设定
告诉 AI 它现在的身份和场景。绝对不要说"你在写冷邮件"，要说"你是一位极其简洁的文字专家"
E
Examples · 示例
提供 1-2 个示例，用 HTML 标签包裹，如 <example>Congrats on joining Stripe as VP Sales</example>。示例是降低 AI 幻觉最有效的方法。
⚠ 绝对不要在 Context 里写"你在写一封冷邮件（cold email）"！AI 训练数据里关于冷邮件的内容大多是质量很差的垃圾模板，一旦触发这个关键词，AI 就会写出浓浓推销味的恶心邮件。应该说"你是一个简洁友好的写作专家"。
实际 Prompt 示例（写邮件标题）
# VARIABLES
{full_name} = 联系人姓名
{recent_activity} = 对方最近的 LinkedIn 动态

# CONTEXT
你是一位极其擅长用简短、精炼语言写作的文字专家。

# INSTRUCTIONS
1. 检查 {recent_activity}，如果有获奖信息，生成以 "Congrats on" 开头的标题
2. 如果没有，检查是否有职位变动，生成祝贺入职/晋升的标题
3. 以上都没有，使用技能认证或工作周年
标题不超过 10 个词，必须第二人称

# EXAMPLES
<example>Congrats on making President's Club this year</example>

AI Formulas · 大白话写公式，完全免费
不用写代码，用自然语言处理数据——抓取、拼接、计数、格式转换都能做
›
是什么
AI Formulas 是 Clay 内置的一个免费功能，专门用来处理表格数据。不用写 Excel 里那些复杂的嵌套公式，用大白话描述你想做什么，AI 自动帮你生成对应的代码并执行。而且这个功能完全免费，不消耗 Clay 积分。
能解决的常见问题
场景1
提取嵌套数组里的数据：从"教育经历"这一列里，提取"最高学历所在学校名称"。这种数据通常是一个嵌套数组，直接用很麻烦，AI 公式一句话搞定
场景2
拼接多列数据：把一个人所有的过往公司名拼成一句话，用于邮件正文里提及对方的经历
场景3
统计数量：计算一个人有几段工作经历，用于打分（工作经历越丰富的高管，决策权通常越大）
场景4
格式转换：把"Apple Inc." → "Apple"，把手机号格式统一，把日期格式转换
✓ AI Formulas 完全免费，不消耗积分，随便用。这是 Clay 里性价比最高的功能之一。
⚡
Conditional Runs · 条件触发执行
设置"如果……才执行"，让工作流更智能，不乱烧积分
›
是什么
瀑布流里"找到邮箱就停止"的逻辑，本质就是条件运行。Conditional Runs 允许你对任何动作设置触发条件——只有满足条件时，这个步骤才会执行，不满足就自动跳过。这是控制成本的核心机制。
配置方法（大白话版）
1
在任意操作的底部，点击 Run Settings
2
点击 Use AI，用大白话输入条件：比如"只有当工作邮箱列为空时，才运行"
3
点击测试生成，AI 自动把你的大白话转成代码条件
真实使用场景
场景：工作邮箱 → 个人邮箱兜底 省积分
你的工作流：先跑工作邮箱瀑布流。然后为个人邮箱瀑布流设置条件："仅当验证过的工作邮箱列为空时才运行"。结果：已经有工作邮箱的 600 人被跳过，只有找不到工作邮箱的 150 人才去找个人邮箱。如果个人邮箱每条查询花 3 个积分，这个条件帮你省下了 600 × 3 = 1800 个积分。

AI Snippets · 碎片化写作，而非一键生成全文
AI 一次只写一句话，多列拼装，让邮件看起来像真人写的
›
最常见的错误做法
很多人想让 AI 一次性生成整封邮件。这几乎不可能写好——AI 任务越宽泛，幻觉越严重，写出来的东西假大空、充满推销味。
正确做法：像顶级 SDR 一样拆分任务
顶尖的销售代表写邮件从不从零开始——他们有一个固定的价值主张模板，只对以下几个关键位置做个性化：
列AI 任务注意事项
列 A生成邮件标题（Subject Line）用 S.P.I.C.E. 框架写 Prompt
列 B生成破冰第一句（First Line）基于对方最近动态/公司新闻
列 C生成衔接第二句（Second Line）必须把列B的内容喂给 AI 作为上文
列 D生成 P.S. 附言轻松、有趣，提高打开率
列 E拼装合并所有碎片用 AI Formula 把上述列拼成最终文案

下周有 15 分钟聊一下吗？

P.S. 我看到你们团队在纽约——我经常去曼哈顿，可以线下见！

Lead Scoring + Data Normalization · 打分 & 数据清洗
对潜在客户打分排序 + 去掉 Inc/LLC 等让邮件看起来像机器人写的后缀
›
Lead Scoring — 是什么
并非所有潜在客户价值相同。Lead Scoring 让你把名单里的每个人打一个综合分，然后销售团队优先跟进高分客户，不把时间浪费在低转化线索上。
打分维度示例
维度条件分数
公司规模 0–500人 / 500–5000人 / 10000人+ 0分 / 1分 / 3分
所在行业 其他 / 广告代理 / 软件公司 0分 / 2分 / 4分
职级 普通员工 / 经理 / 总监+ / C-Suite 0分 / 1分 / 2分 / 3分
最高可得 10 分。你可以只联系 7 分以上的潜在客户，效率提升数倍。
Data Normalization — 公司名称清洗
从数据库抓回来的公司名经常是 "Apple Inc." "Tesla Motors LLC" "Google Holdings Co., Ltd."。如果把这些名字直接放进邮件里，收件人一眼就看出是机器群发的，立刻删除。
操作
Enrichment 面板 → Tools → Normalize → Normalize company names，选中公司名列，点运行。"Apple Inc." → "Apple"，一键完成，免费。
✓ Native Tools 里还有很多免费的数据处理工具：从文本提取 URL、对数据计数 / 排名 / 分组，等等。建议新手花 10 分钟探索一下这个区域。

E · Export — 推送与输出

3 个功能
🏦
Push to CRM · 推送到客户管理系统
两步验证法，先查重再新建，避免 CRM 变垃圾场
›
最常见的灾难性错误
新手会直接把 Clay 表格里的所有人一键推送到 HubSpot/Salesforce，创建新联系人。结果：同一个人出现了 3 条重复记录，销售团队不知道该用哪条，给同一个客户发了 2 封相同的邮件，显得极其不专业，甚至被拉黑。
正确方法：两步验证法
步骤 A
先查：这个人已经在 CRM 里了吗？
用工作邮箱在 CRM 里搜索对方。邮箱是每个人唯一的标识符。这一步得到结果：已存在 / 不存在。
→
步骤 B（条件触发）
仅当"不存在"时，才新建联系人
如果步骤 A 发现此人已在 CRM，跳过。只有全新联系人才触发"Create Contact"操作。
真实使用场景
场景：每周自动同步新线索到 HubSpot 数据卫生
你每周用 Clay 找 200 个新联系人，用两步验证法推送到 HubSpot。其中 50 人已经在 HubSpot 里（之前已经联系过），自动跳过。只有 150 个真正新的联系人被创建，CRM 保持干净，销售团队的工作效率也大幅提升。

你在 Clay 里用 AI 生成了 3 封邮件序列的所有个性化内容：邮件1标题、邮件1正文、邮件2标题（Follow-up）、邮件2正文……把这些全部映射为 Custom Variables 推送到 Instantly。在 Instantly 里，你用 {SubjectLine1}、{EmailBody1}、{SubjectLine2} 等变量引用这些内容。结果：每个联系人收到的每一封邮件都是完全个性化的，但全程零人工操作。

基础设施 — 平台底层能力

3 个功能
🔗
Write to Table · 跨表数据流转
把嵌套在一个格子里的多条数据，展开到新表格，实现跨表联动
›
用大白话解释
你用 Lookalikes 功能，给 162 家公司各找了 10 家相似公司。结果是：每个格子里塞了 10 家公司的数据——162 个格子，每个格子里有 10 家。这 1620 家公司没法直接用。Write to Table 就是把每个格子里的 10 家公司"展开"，让每家公司独占一行，写进新的表格里。
4 个配置步骤
1
选择目标表格：新建表格，还是写入已有表格。通常选新建，保持数据清晰。
2
选择要展开的列表：选择那个带有"数字+大括号"图标的列（说明里面有多条数据），如"Lookalike Companies"列。
3
提取具体字段：告诉 Clay 你要把嵌套数据里的哪些字段变成新表格的列，如公司名、域名、员工数、行业。
4
保留来源标记：勾选"Add other data from this table"，把原始公司名也带过去，方便知道"这家新公司是因为哪家老客户找出来的"。
隐式使用场景
你在公司表里点击"Find People at these companies"，系统其实就是在后台自动跑了一个 Write to Table，把每家公司找到的多个人展开成独立的行，写进了人员表。两张表通过 Lookup 列自动保持同步。

HTTP API · 接入任意第三方工具
Clay 没有原生集成的工具，只要有 API 就能接进来
›
是什么
API 是互联网上软件之间对话的语言。只要一个工具提供了开放 API（绝大多数 SaaS 工具都提供），你就可以在 Clay 里通过 HTTP API 功能把它接进来，不需要懂代码，不需要复杂认证，通常只需要粘贴一个 API Key。
两种请求类型
类型作用示例
GET 请求从外部系统"拿"数据从 Yelp 获取某个地址附近评分最高的餐厅列表
POST 请求向外部系统"发送"数据并触发操作把图片 ID 和文字发给 Imgflip，生成一张定制梗图，返回图片链接
降低门槛的 3 个辅助功能
1
预设模板：Clay 提供了常用 API 的配置模板，直接套用
2
AI 辅助配置：把 API 文档的网址扔给 Clay，内置 AI 自动帮你读文档、填好参数
3
可复用配置：配置好一个 API 之后，保存为个人模板，在任何其他工作簿里一键复用
真实使用场景
场景：在开发信里插入对方公司附近的餐厅推荐 超级个性化
你卖商业地产服务，目标客户是纽约的企业。你用 Yelp API（GET 请求），把每家目标公司的办公地址传进去，让 Yelp 返回该地址附近评分最高的晚餐餐厅。然后邮件里写："Hi [姓名]，下周我刚好去你们办公室附近的 [Yelp 返回的餐厅名] 吃饭，有空一起聊聊贵公司的商业地产需求吗？"——这封邮件的回复率远超任何泛泛的推销。

Workbook & Table · 数据组织结构
先建 Workbook（工作簿），再在里面建多张 Table（表格）
›
关系类比
Workbook 就像 Excel 文件，Table 就像 Excel 里的多个工作表。一个 Workbook 里可以有多张 Table，这些 Table 之间可以通过 Lookup 列互相关联，数据自动同步。
标准的三表结构（最佳实践）
表1
Companies Table：存放所有目标公司及其数据（行业、规模、融资阶段、技术栈……）
表2
People Table：存放从这些公司里找出的决策人（自动与公司表联动），含邮箱、职位、个性化文案
表3
Lookalikes Table（可选）：从现有客户扩展出来的相似公司名单
✓ 永远先建 Workbook 再建 Table，不要直接建孤立的 Table。Workbook 让多表之间的关联和管理变得清晰，尤其当工作流变得复杂时，你会感谢这个习惯。
